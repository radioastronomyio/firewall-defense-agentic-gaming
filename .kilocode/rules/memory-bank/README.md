# Memory Bank

KiloCode context files for Grid Defense RL. These files are read by AI agents at session start to establish project context.

---

## Contents

| File | Purpose | Update Frequency |
|------|---------|------------------|
| [brief.md](brief.md) | Project identity, purpose, audience | Rarely |
| [architecture.md](architecture.md) | System structure, data flow, key decisions | When architecture changes |
| [context.md](context.md) | Project history, multi-model synthesis notes | As project evolves |
| [product.md](product.md) | Vision, deliverables, success metrics | When goals change |
| [tech.md](tech.md) | Stack, dependencies, constraints | When tech changes |
| [tasks.md](tasks.md) | Current work, backlog, phase status | Frequently |

---

## Usage

KiloCode loads these files automatically. Human contributors can reference them for project onboarding.

---

## Maintenance

- **brief.md** — Update only when fundamental purpose changes
- **architecture.md** — Update when directory structure or data flow changes
- **context.md** — Append to "Current Status" as phases complete
- **product.md** — Update when deliverables or success metrics change
- **tech.md** — Update when dependencies or constraints change
- **tasks.md** — Update at each session to reflect current work
